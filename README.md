# Quiz App

ეს არის მრავალფუნქციური Android აპლიკაცია, რომელიც საგანმანათლებლო და ინტერაქტიულ ქვიზებს აერთიანებს. პროექტის მიზანია, მომხმარებელს შესთავაზოს ცოდნის გაღრმავების საინტერესო გზა სხვადასხვა სფეროში — პროგრამირებიდან (C#, Android არქიტექტურა) ისტორიასა და გეოგრაფიამდე.

## 🚀 ძირითადი ფუნქციები

*   **ავტორიზაცია & რეგისტრაცია:** მომხმარებლის პირადი ანგარიშის მართვა Local DB-ს გამოყენებით.
*   **ქვიზის კატეგორიები:** სამი ძირითადი მიმართულება: პროგრამირება (C#, Android), ისტორია და გეოგრაფია.
*   **დინამიური ქვიზის პროცესი:**
    *   კითხვების თანმიმდევრობის შეცვლა (shuffled) თანმიმდევრობა.
    *   30-წამიანი ტაიმერი თითოეულ კითხვაზე.
    *   პასუხების შემოწმება (Review Answers) ქვიზის დასრულების შემდეგ.
*   **სტატისტიკა & პროფილი:** მომხმარებლის პირადი მიღწევების (ჩატარებული ქვიზების რაოდენობა, საშუალო ქულა) ვიზუალიზაცია.
*   **ლიდერბორდი:** გლობალური რეიტინგი, სადაც ჩანს საუკეთესო შედეგების მქონე მომხმარებლები.
*   **მონაცემთა ავტომატური განახლება:** სპეციალური ლოგიკა კითხვების ტექსტების ავტომატური სინქრონიზაციისთვის

## ტექნოლოგიები

*   **Architecture:** MVVM (Model-View-ViewModel)
*   **UI:** ViewBinding, Material Design 3, Navigation Component
*   **Asynchrony:** Kotlin Coroutines & Flow

## მონაცემთა ბაზა და კავშირი 

აპლიკაციაში მონაცემთა შენახვა და მართვა ხორციელდება **Room Persistence Library**-ის გამოყენებით, რომელიც SQLite-ის ზემოთ არსებულ აბსტრაქციას წარმოადგენს. ბაზასთან კავშირი დაყოფილია შემდეგ ფენებად:

1.  **Room Database (`AppDatabase`):** მთავარი წერტილი, რომელიც აკავშირებს აპლიკაციას SQLite ფაილთან. შექმნილია *Singleton* პატერნით, რათა თავიდან ავიცილოთ რესურსების ზედმეტი ხარჯვა.
2.  **DAO (Data Access Object - `QuizDao`):** ინტერფეისი, სადაც განსაზღვრულია ყველა SQL მოთხოვნა (Query). აქ ვიყენებთ `Flow`-ს მონაცემების რეალურ დროში მისაღებად და `suspend` ფუნქციებს ასინქრონული ჩაწერისთვის.
3.  **Repository Pattern (`QuizRepository`):** შუამავალი ფენა DAO-სა და ViewModel-ს შორის. ის უზრუნველყოფს მონაცემთა წყაროს იზოლაციას და მართავს **Prepopulate** ლოგიკას (ბაზის საწყის შევსებას).
4.  **Fixed IDs Strategy:** მონაცემთა განახლებისთვის კითხვებს აქვთ ფიქსირებული ID-ები. `OnConflictStrategy.REPLACE`-თან კომბინაციაში, ეს საშუალებას გვაძლევს განვაახლოთ კითხვების ტექსტი მომხმარებლის შედეგების წაშლის გარეშე.
